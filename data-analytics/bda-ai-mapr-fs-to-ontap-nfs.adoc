= MapR-FS para ONTAP NFS
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Esta seção fornece as etapas detalhadas necessárias para mover dados do MAPR-FS para o ONTAP NFS usando o NetApp XCP.

. Provisione três LUNs para cada nó do MAPR e dê aos LUNs a propriedade de todos os nós do MAPR.
. Durante a instalação, escolha LUNs recém-adicionados para discos de cluster do MAPR usados para o MAPR-FS.
. Instale um cluster do MAPR de acordo com a documentação do MAPR 6,1.
. Verifique as operações básicas do Hadoop usando comandos MapReduce, `hadoop jar xxx` como .
. Mantenha os dados do cliente no MAPR-FS. Por exemplo, geramos aproximadamente um terabyte de dados de amostra no MAPR-FS usando Teragen.
. Configure o MAPR-FS como exportação NFS.

. Atribua um intervalo IP virtual a um servidor específico ou a um conjunto de servidores no cluster do MAPR. Em seguida, o cluster MAPR atribui um IP a um servidor específico para acesso a dados NFS. Os IPs habilitam alta disponibilidade, o que significa que, se um servidor ou rede com um IP específico sofrer falha, o próximo IP do intervalo de IPs pode ser usado para acesso NFS.
+

NOTE: Se você quiser fornecer acesso NFS a partir de todos os nós do MAPR, poderá atribuir um conjunto de IPs virtuais a cada servidor e usar os recursos de cada nó do MAPR para acesso a dados NFS.
